CFI grade sheet entry

  1. Goal: Allow CFIs to input grades from ground and flight events
  2. It should look like the paper grade sheet in formatting. We can adjust either one as necessary
  3. All of the values should be pre-filled for the instructor to minimize the admin burden on them. They should be given a set of links upon logging in for each event they're scheduled to have finished that does have any values in the grade sheet.

Demo

Grade sheet entry for Midn W. T. "Trog" Door on event C2

Tail number:*This should be a dropdown listing all active plane tail numbers. plane->tail_number
Instructor name:*This should be a dropdown listing rank, first_name, last_name of everyone who is active in the CFI table. user[cfi_id]->title,first_name,last_name
Student name:*This should be a dropdown listing rank, first_name, last_name of everyone who is active in the student table.
Event:**This should be a dropdown listing the name of each event in the event table. event->name
Outcome:*('canceled','fail','incomplete','scheduled','marginal','pass')
Reason for not completing:*Should appear if incomplete or canceled is selected above. Current possible list of cancelations: weather, maintenance, plane_availability, instructor, student, wrong event, wrong instructor,crew rest,other. These will be in the cancel table: cancel->reason
Takeoff time:Preloaded from the sortie->scheduled_takeoff. It gets saved as student_sortie[sortied_id,student_id]->actual_takeoff
Land time:Preloaded from the sortie->land. It gets saved as student_sortie[sortied_id,student_id]->actual_land
Dual flight time:Preloaded from the event->dual_flight_hours. It gets saved as student_sortie[sortied_id,student_id]->dual_flight
Solo flight time:Preloaded from the event->solo_flight_hours. It gets saved as student_sortie[sortied_id,student_id]->solo_flight
Instructional time:Preloaded from the event->instruction_hours. It gets saved as student_sortie[sortied_id,student_id]->total_instruction_time
Dual landings: Preloaded from the event->default_dual_landings. It gets saved as student_sortie[sortied_id,student_id]->dual_landings
Solo landings: Preloaded from the event->default_solo_landings. It gets saved as student_sortie[sortied_id,student_id]->solo_landings
